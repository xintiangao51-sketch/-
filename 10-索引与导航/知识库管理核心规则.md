# 📘 知识库管理核心规则

> 最后更新：2026-06-22 | 版本 v2.0

## 一、三层知识管理架构

```
┌─────────────────────────────────────────────┐
│  L1 · 捕获层 (Capture)                       │
│  Quick capture → 00-工作台 Inbox → 每日笔记    │
│  工具：Templater 模板 + 快捷键一键捕获          │
├─────────────────────────────────────────────┤
│  L2 · 组织层 (Organize)                      │
│  PARA 分类 + MOC 地图 + Dataview 动态索引      │
│  原则：原子化笔记 + 双向链接 + 渐进式整理        │
├─────────────────────────────────────────────┤
│  L3 · 产出层 (Create)                        │
│  AI 辅助提炼 → 知识沉淀（KC卡片）→ 项目交付物    │
│  工具：Claudian (Claude) + Excalidraw + Kanban │
└─────────────────────────────────────────────┘
```

### L1 捕获层 — 零摩擦入库

| 来源 | 入口 | 工具 |
|:-----|:-----|:-----|
| 日常灵感/问题 | `00-工作台/` 快速笔记 | Templater 模板 + 快捷键 |
| 网页剪藏 | 浏览器选中 → Claude → 写入笔记 | Claudian Agent |
| 项目文件 | 拖入对应项目目录 | Obsidian 文件管理器 |
| AI 对话导出 | Hermes/ChatGPT 对话 → 提炼要点 | Hermes → MEMORY.md |
| 信息价/政策 | 爬虫脚本 → 自动写入 | Local REST API + 定时任务 |

### L2 组织层 — PARA + MOC

| PARA 象限 | 00-99 映射 | 说明 |
|:----------|:----------|:-----|
| **P**rojects | `09-项目资料/`、`00-工作台/` | 活跃项目 + 驾驶舱 |
| **A**reas | `01-造价核心/`、`02-业务中台/`、`03-合同与法务/`、`04-施工方案/`、`05-投标报价/` | 持续负责的领域 |

### L3 产出层 — 从信息到交付

```
原始数据 ──→ 分析笔记 ──→ 知识卡片 (KC) ──→ 交付物
```

## 二、四大核心方法论

| 方法论 | 来源 | 核心逻辑 | 适用场景 |
|:-------|:-----|:---------|:---------|
| **PARA** | Tiago Forte | 项目/领域/资源/归档 四分类 | ✅ 00-99 目录结构 = PARA 变体 |
| **Zettelkasten** | Luhmann | 原子化笔记，每张卡一个概念 | ✅ 造价知识点拆成独立 KC 卡片 |
| **MOC** | Nick Milo | 地图笔记，链接聚合 | ✅ `10-索引与导航/` = MOC 专区 |
| **IMR** | Nick Milo | 构思→制作→关联 创作流程 | ✅ 方案比选/投标书适用 |

### PARA 落地规则

```
P → 09-项目资料/<项目名>/        ← 每个项目一个目录
A → 01~05（按造价业务域分）       ← 持续维护，不分时间
R → 06~08 + 11~14（按资源类型分）  ← 被动积累，定期整理
Ar → 99-归档库/                   ← 项目完结后移入
```

### Zettelkasten 落地规则

1. 每张 KC 卡片 = 一个独立知识点
2. 卡片命名：`KC-XXX_主题描述.md`
3. 必须包含：问题场景 → 分析 → 结论 → 来源 → 关联卡片
4. 通过双向链接形成知识网络

### MOC 落地规则

1. `10-索引与导航/` 下的笔记只做聚合，不放内容
2. 每个一级目录有 `总览.md` 或 `README.md` 作为迷你 MOC
3. 首页 `🏠 知识库首页.md` 是全库根 MOC

## 三、Templater — 标准化模板引擎

### 模板清单

| 模板 | 位置 | 用途 | 触发方式 |
|:-----|:-----|:-----|:---------|
| 造价笔记模板 | `.obsidian/templates/造价笔记模板.md` | 定额/清单/变更分析 | 新建笔记 → Templater 插入 |
| 项目记录模板 | `.obsidian/templates/项目记录模板.md` | 新建项目总览 | 同上 |
| 项目元数据模板 | `.obsidian/templates/项目元数据模板.md` | 项目 frontmatter | 同上 |
| 文件夹索引模板 | `.obsidian/templates/文件夹索引模板.md` | 目录索引 | 同上 |
| **项目总览模板** | `00-工作台/Templates/项目总览模板.md` | 增强版项目总览 | Templater 选择 |
| **造价分析模板** | `00-工作台/Templates/造价分析模板.md` | 造价分析（含新疆系数） | Templater 选择 |
| **合同审查模板** | `00-工作台/Templates/合同审查模板.md` | 合同风险审查 | Templater 选择 |

### 模板核心字段标准

```yaml
---
# 必填
tags: [领域/子域, 属性]    # 最多5个，层级优先
created: YYYY-MM-DD        # 自动生成
type: index | knowledge-card | project | reference

# 可选（按模板类型）
project_name: 项目名称
contract_amount: 合同金额
status: 在建 | 竣工 | 投标中 | 结算
---
```

## 四、Excalidraw — 造价场景用法

### 场景 → 图表类型

| 造价场景 | Excalidraw 图表 | 示例 |
|:---------|:---------------|:-----|
| 施工工艺 | 流程图（箭头+矩形） | 钢筋绑扎→支模→浇筑→养护 |
| 索赔事件 | 时间线（水平线+事件节点） | 延误→通知→签证→审批→结算 |
| 方案比选 | 决策树（菱形判断+分支） | 桩基 vs 独立基础 → 成本/工期对比 |
| 合同关系 | 关系图（方框+连线） | 甲方→总包→分包→供应商 |
| 取费链路 | 层级图（嵌套框） | 人工+材料+机械→管理费+利润→规费→税金 |

### 使用规则

1. **嵌入而非独立** — Excalidraw 图直接 `! <!-- example: replace with actual file -->` 嵌入笔记中
2. **轻量优先** — 10 个元素以内的图，不用 Visio/ProcessOn
3. **AI 辅助** — 用 Claudian 生成 Mermaid → Excalidraw 转换

## 六、知识库工作流（多 Agent 分工）

| 任务类型 | Agent | 操作方式 |
|:---------|:------|:---------|
| **深度分析**（合同审查、方案比选、索赔论证） | Claude (Claudian) | Obsidian 中 @Claudian → 读文件 → 分析 → 写回笔记 |
| **批量执行**（Excel 处理、定额套用、图纸提量） | Codex | Obsidian 中切到 Codex provider → 批量脚本 |
| **记忆沉淀**（项目总结、经验提炼、方法论） | Hermes | 对话 → 生成草案 → 确认 → 写入 MEMORY.md |
| **快速查询**（材料价、定额编号、规范条文） | Hermes (主对话) | 直接问，结合联网搜索 |
| **可视化**（工艺草图、索赔时间线、决策树） | Excalidraw | 笔记内嵌，AI 辅助生成 |
| **进度追踪**（投标阶段、项目节点） | Kanban | Obsidian 看板，拖拽更新 |

## 七、知识卡片 (KC) 标准

### 卡片模板

```markdown
---
tags: [领域标签, 属性标签]
created: YYYY-MM-DD
source: 来源URL或文件路径
status: 已核实 | 待核实 | 草稿
related: [KC-xxx, KC-yyy]
---

# KC-XXX: 标题

## 问题场景
一句话描述什么情况下需要这个知识。

## 核心结论
最关键的 1-3 条结论。

## 依据/来源
- 法规/规范/文件名称
- 网址或路径

## 实操指引
- 对造价工作的具体影响
- 行动建议

## 关联卡片
- KC-xxx <!-- placeholder -->
-  <!-- example: replace with actual KC number -->
```

### 卡片编号规则

- **KC-001~099**：政策法规与标准
- **KC-100~199**：定额与计价
- **KC-200~299**：工程量计算
- **KC-300~399**：材料设备价格
- **KC-400~499**：合同与法务
- **KC-500~599**：施工技术
- **KC-600~699**：投标报价策略
- **KC-700~799**：案例复盘
- **KC-800~899**：AI 工具与技能
- **KC-900~999**：方法论与知识管理
