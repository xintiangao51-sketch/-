## 三、变更影响面速查

### 3.1 改 SOUL.md / AGENTS.md 时

```
SOUL.md    → 影响所有 Agent 的身份识别
AGENTS.md  → 影响所有 Agent 的公共行为
MEMORY.md  → 影响记忆工程所有操作
变更评估.md → 影响变更决策流程
```

**改前必读**：5 篇 Agent 工作手册 + 变更评估.md。

### 3.2 改工作手册时

```
知识库运行手册.md → 影响所有 Agent 的执行规则
知识库结构.md     → 影响 FM 字段、目录结构、索引格式
系统架构.md       → 本文件，影响关联图
```

**改前必读**：所有 4 Agent Profile。

### 3.3 改 🏷️ 标签体系.md 时

```
标签体系.md → 影响所有页面 tags
           → 影响 Dataview 查询
           → 影响 lint_xinjiang.py 的标签统计
```

**改前必读**：`D:\知识库\.scripts\lint_xinjiang.py` 中 `造价相关` 列表是否需要同步。

### 3.4 新建丹房页（提炼完成）时

```
1. 原料 → 标注 已提炼 + 写回链
2. 知识库总索引.md（KC-480）追加新行
3. 日志.md 追加一行
4. Git commit
（交叉链接由 每日检 18:00 执行）
```

### 3.5 改动任何文件名/路径时

```
1. 文件本身改名
2. 知识库总索引.md 更新引用
3. 知识库结构.md（如目录描述有变）
4. 全库 [[10-索引与导航/索引与导航-总览]] → 改为 [[10-索引与导航/索引与导航-总览]]
5. 日志.md 记录
```

```

□ 这是删旧文件？ → 总索引删引用 + 结构.md 删目录行？
□ 这次涉及 FM？ → 规范见 知识库结构.md §二
□ 这次涉及回链？ → 放在 frontmatter 后、正文前，独占一行
□ 最后检查：日志.md 写了吗？git commit 了吗？
```
