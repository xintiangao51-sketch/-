# 知识库运行手册

> 灵台运行手册（v2026-06-12）改造吸收版，造价工程师场景本地化。
> 本文件定义 D:\知识库 的完整运行规范——从理念到流程到约束，平台无关。
> 4 Agent（Hermes / Claude / Codex / Workbuddy）在此基础上注入各自执行细节。

---

## 一、身份与职责

D:\知识库 的整理引擎。核心职责是把零散的造价资料（合同/签证/定额/价格/法规）转化为结构化知识条目，并持续维护知识库的健康。

**不编造**：不得编造定额/价格/图纸尺寸/合同条款。
**不润色原文**：不擅自修改原料（合同、签证单）措辞。
**但具有通用价值的综合回答可存为知识条目**（写回链路）。

---

## 二、核心理念

| 理念 | 落地机制 |
|------|---------|
| 原料只存不改 | `09-项目资料/` 是真理来源 |
| 知识库由 AI 增量维护 | 01-07/14 等知识层 |
| 累积而非检索 | 每次加入新资料，整合进已有结构 |
| 不重复不漂移 | 已有覆盖的不再提炼，参数保持一致 |

## 三、三大操作

### 3.1 提炼（ingest）

把原始资料转化为结构化知识条目。

**流程**：

```
原料 → 读/判断
  │
  ├── 已有覆盖 → 标记 已包含，结束
  │
  ├── 纯结构/流程产物 → 标记 已跳过，结束
  │
  └── 有新概念/新内容 → 按 L1/L2/L3 分流：
        ├── L1 自动放行：纯事实更新、人工单价按信息价刷新
        ├── L2 摘要确认：已知框架新角度、≤7篇原料
        └── L3 新概念提炼：新定额、新法规、新方法论
```

**统计原料数量**：记录本次提炼使用的原料文件数，作为 frontmatter `原料数量` 的值。

**写入原则**：tmp → mv 原子操作（崩了可重跑，不产脏数据）。

### 3.2 问知（query）

查询知识库中的已有内容。

- 全库 grep 标题找相关页面（不读索引，不猜域）
- 定位相关页面，综合回答
- 引用具体页面
- 有通用价值的回答可问用户是否存为知识条目

### 3.3 体检（lint）

定期检查知识库健康。

```
D:\知识库\.scripts\lint_xinjiang.py
```

分两层：
- **入库体检**：每次提炼后执行——断裂链接、旧路径、frontmatter、标签合规
- **定期巡检**：死胡同、孤立页面、标签分布

所有修复经用户确认，**不自动修复**。

## 四、核心流程

### 4.1 三级确认制 L1/L2/L3

| 等级 | 适用场景 | 确认 | 造价场景示例 |
|:--:|-------|:--:|------|
| **L1 自动放行** | 纯事实/数据更新、已知框架扩展 | ❌ | 人工单价按信息价刷新、定额子目微调 |
| **L2 摘要确认** | 已知框架新角度、≤7篇原料 | ❌ | 同一项目同类签证整理入册 |
| **L2.5 批量摘要** | ≥8篇原料浇灌同一域 | ⚠️ | 全年签证台账季度汇总 |
| **L3 新概念提炼** | 新概念/新框架引入 | ✅ | 新疆新发布定额子目入库 |
| **框架变更/台律修改** | 核心规则修订 | ✅ | 取费口径大改、税率调整 |
| **矛盾检测** | 跨页断言冲突 | ❌ 只记录 | 签证台账与合同条款冲突 |
| **台律修改** | 影响所有操作 | ✅ | 五件套文件变更 |

**执行细则**：
- L1/L2 直接执行，矛盾/语义只记录不确认
- 涉及造价数据/费率/合同条款时，自动升级为 L3
- 框架变更前必须输出变更影响面（参照 [[10-索引与导航/AI-Agent五件套/变更评估|变更评估]]）

### 4.2 提炼后必须执行

每轮提炼写入完成后，不可跳过。

| 步骤 | 检查内容 | 产出 |
|------|----------|------|
| 交叉链接 | 新页推荐阅读链出（新→旧）| 3-6 条带关系说明 |
| 原料回链 | 在每篇源原料正文末尾追加 `→ [[10-索引与导航/索引与导航-总览]]` | 原料→知识库反链 |
| 更新索引 | `知识库总索引.md` 追加新行 | 全量目录自动增长 |
| 选题判断 | 本次提炼产生的新角度是否符合选题标准 | 必要时记入选题池 |
| 日志追加 | `日志.md` 追加一行 | 时间/类型/摘要/关联 |
| Git 提交 | 先写日志 → 校验 → 提交 | 不写日志不提交 |

### 4.3 体检定期巡检

| 检查项 | 说明 | 频率 |
|--------|------|:--:|
| 断裂链接 | 所有 [[10-索引与导航/索引与导航-总览]] 目标是否存在 | 每日 |
| 死胡同 | 入链=0 但出链>0 | 每日 |
| 孤立页面 | 入链=0 且 出链=0 | 每日 |
| 标签分布 | 标签词频统计 | 每日 |
| 矛盾检测 | 跨页断言冲突 | 每日 |
| 时效提醒 | 有时效字段 + 日期 > 90 天 | 每月 |
| 语义探针 | bge-small-zh 相似度 ≥ 0.75 未链接候选 | 每周 |

### 4.4 写回链路（造价场景特别重要）

技能产出有价值的结论/分析/发现时，直接按提炼流程提炼为知识页。**不经过 09-项目资料/**——好答案跟外部原料一样，直接进 知识库。

**造价场景示例**：
- 审计一份签证单发现争议 → 写"签证争议处理 SOP" → 入 `01-造价核心/变更索赔/`
- 投标报价过程中发现策略 → 写"新疆房建报价策略" → 入 `05-投标报价/`
- 复核工程量时发现图纸疑点 → 写"图纸提量注意清单" → 入 `01-造价核心/工程量/`

**FM 规则**：写回产物不加 `品级` 和 `原料数量`（属于提炼体系），加 `来源: 写回` 作为标记。

### 4.5 上下文冲突优先级

| 层 | 说明 | 优先级 |
|:--|:----|:-----:|
| 规则层 | 五件套 SOUL/AGENTS/MEMORY | 最高 |
| 知识层 | 01-07 + 14 知识页 | 中 |
| 档案层 | 09-项目资料 + 11-材料价格库 | 最低 |

当 LLM 感知上下文过载时，按档案层→知识层→规则层 逆序丢弃。

## 五、硬性约束

1. **不编造**：所有内容来自原料原文，信息不足写"原文未提及"或标 ⚠待核实
2. **不删原文**：原料文件永远不修改内容
3. **不覆盖**：同名条目合并，冲突部分保留来源+时间
4. **不评价**：不对观点做对错判断
5. **不遗漏来源**：每个条目标注原始来源
6. **不自动删除**：只标记状态，不删除文件
7. **不自动修复**：体检发现的任何问题必须经用户确认再改
8. **台律修改需确认**：修改 五件套 文件前，必须先列出变更摘要，等用户确认后再执行

## 六、与用户的分工

| 用户（昕田） | AI（老高） |
|------|-----|
| 提供原始资料（合同/签证/价格/定额） | 读取、清洗、整理 |
| 引导讨论方向 | 写/更新条目、维护交叉引用 |
| 判断什么重要 | 体检、回答查询 |
| 决定组织方式 | 不替代用户做价值判断 |

## 七、台律迭代制

每次发现 AI 输出问题，不修单次提示词，而是提炼为可复用规则补充到规范文件。规范一旦更新，后续所有操作自动遵守新规则。

### 触发条件

1. 连续 3 次对同类内容判断失误 → 补充判断规则
2. 用户纠正 AI 的某个行为模式 → 提炼为可复用约束
3. 新增标签类型 → 更新 [[🏷️ 标签体系|🏷️ 标签体系]]
4. 体检发现系统性问题 → 补充预防规则

### 提炼为规则后的判定

- 行为准则 → 更新本文件
- 格式规范 → 更新 [[10-索引与导航/AI-Agent五件套/工作手册/知识库结构]]
- 决策判断 → 更新 [[10-索引与导航/AI-Agent五件套/变更评估|变更评估]]

### 试运行案例：KC-130 拆页（2026-06-25 首次实战）

> 背景：D 档对 `03-合同与法务/变更签证实战指南.md`（727 行 / 9 节）做枢纽页+辐页拆页试运行。

**3 条沉淀经验**（来源：写回链路 §4.4）：

| 经验 | 提炼规则 | 写入文件 |
|------|---------|---------|
| 拆页前先用 lint 跑一遍 | 拆页前必跑 `lint_xinjiang.py` 扫描 `>100行` 或 `>7节` 页面作为候选 | 本文件 |
| 占位页创建前先 grep 是否已存在 | 创建占位页前必须 grep 目标文件名，避免重复建 | 本文件 |
| 占位页暴露二级断链是健康信号 | 每建一个占位页暴露 2-3 条新断链 = 知识图谱在立体化，不是 bug | 本文件 + lint_xinjiang.py 注释 |

**SOP（拆页闭环 5 步）**：

```
1. 探查候选
   └─ 运行：python D:\知识库\.scripts\lint_xinjiang.py
   └─ 找出：行数>100 或 节数>7 的页面

2. 选主题
   └─ 优先选 C 档主题直接命中、C1/C5 验证过的页面

3. 拆页
   └─ 备份原版为 _原版_YYYYMMDD.md
   └─ 枢纽页：60-80 行（核心定义 + 速览表 + 框架地图 + 推荐阅读）
   └─ 辐页：每页 80-160 行（按主题拆，原章节标题保留为 ## 三级标题）

4. 闭环断链
   └─ 跑 lint 找推荐区 GAP
   └─ 按 L1/L2/L3 分级处理：
      ├─ L1 简单占位 → 直接建 30-50 行占位页
      ├─ L2 重要内容 → 建 50-100 行卡片
      └─ L3 已存在 → 删除我新建的重复页

5. 更新索引 + 日志
   └─ KC-480 总索引追加新行
   └─ 00-工作台/操作日志.md 追加 1 条
```

**试运行验证结果**：

- ✅ C1 枢纽页+辐页架构：60 行枢纽 + 4 个聚焦辐页，符合规范
- ✅ A2 三级确认制：占位页创建自动升级 L3
- ✅ C3 写回链路：本案例 3 条经验可写回本文件
- ✅ C5 台律迭代制：拆页→暴露断链→创建占位页 = 完整迭代闭环
- ⚠ 雪球效应：每建占位页暴露 2-3 条二级断链，是健康信号

### 试运行案例 2：语义探针 + 去重（2026-06-25 首次实跑）

> 背景：E3 档用 TF-IDF 版语义探针扫描 887 文件，识别 24086 对高相似度页面。

**沉淀经验**（来源：写回链路 §4.4）：

| 经验 | 提炼规则 | 写入文件 |
|------|---------|---------|
| 语义探针 24086 对中仅 ~100 对有真实价值 | 相似度 ≥ 0.60 才需处理，0.35-0.59 大部分是模板同质化噪音 | 本文件 |
| 5 个 KC 重复卡全是自动提取的"待补充"占位 | 自动提取占位卡不分配连续 KC 编号，首次落地后再编号 | 本文件 + KC 编号规范 |
| TF-IDF 精度不足但零下载可用 | 首次用 TF-IDF 快速扫，定期用 bge-small-zh 精扫 | semantic_scan_tfidf.py |

**去重结果**：

| 组 | 文件 | 操作 | 状态 |
|:--:|------|:----:|:---:|
| KC-370/371 造价实战经验库 | 删除 371，保留 370 | ✅ |
| KC-280/281 材料设备价格趋势 | 删除 281，保留 280 | ✅ |
| KC-220/221 国源电力报价策略 | 删除 221，保留 220 | ✅ |
| KC-600/601 投标报价策略库 | 删除 601，保留 600 | ✅ |
| KC-131 变更签证实战指南 | 删除 131（v1.0 版，已被 v2.0 替代）| ✅ |
| 乌鲁木齐2026-04 速查 | 合并：删除近期材料价副本，保留材料价格版 | ✅ |

**去重 SOP（5 步）**：

```
1. 跑探针
   └─ python D:\知识库\.scripts\semantic_scan_tfidf.py (或 semantic_scan_xj.py)

2. 解读结果
   └─ ≥ 0.90 = 完全相同（副本/备份）
   └─ 0.75-0.89 = 高度相似（可考虑合并）
   └─ 0.60-0.74 = 建议审查
   └─ < 0.60 = 模板同质化噪音，忽略

3. 去重执行
   └─ 先 Read 两个文件确认内容
   └─ 完全相同 → 保留低编号，删除高编号
   └─ 部分差异 → 合并后保留完整版

4. 旧文件标注
   └─ 在保留文件的 frontmatter 后加"重复处理"段
   └─ 索引删除被删行

5. 日志 + 确认
   └─ 操作日志追加 1 条
   └─ 跑 lint 验证文件计数
```

## 九、质量评估定期化（P2-8 · 2026-06-25 新增）

质量评估应定期执行，利用 `lint_xinjiang.py` 自动化扫描，确保知识库健康度不退化。

### 周期

| 周期 | 执行内容 | 产出 | 执行者 |
|------|---------|------|:----:|
| **每次会话** | 提炼/写回/变更操作后随手跑一次 lint | — | 老高 |
| **每周一** | 完整 lint 扫描 + 标签检查 + 死胡同趋势 | `操作日志.md` 记录结果 | 老高 |
| **每月 1 日** | 语义探针扫描 + 时效窗口超 90 天检查 | 知识库质量评估报告更新 | 老高 |

### 自检命令

```bash
# 每周一基准
python D:\知识库\.scripts\lint_xinjiang.py

# 正文 #主标签 检查
python D:\知识库\.scripts\check_body_tags.py --list

# 语义探针（如已安装 sentence-transformers）
python D:\知识库\.scripts\semantic_scan_tfidf.py

# 去重检查
python D:\知识库\.scripts\semantic_scan_tfidf.py --threshold 0.9
```

### 健康度指标

| 指标 | 健康 | 警告 | 需行动 |
|------|:----:|:---:|:-----:|
| 缺口数 | < 150 | 150-200 | > 200 |
| 索引缺口 | < 100 | 100-150 | > 150 |
| 推荐区 GAP | < 20 | 20-50 | > 50 |
| FM tags 渗透率 | > 90% | 70-90% | < 70% |
| 正文 #主标签 渗透率 | > 30% | 10-30% | < 10% |

### 死胡同分类指南

死胡同（有出无入）不全是坏事。实际诊断中发现：

| 类型 | 示例 | 处理 |
|------|------|------|
| DDC 卡片 | `06-参考文档/DDC数字驱动建设/cards/` | **设计预期**，忽略 |
| SKILL 脚本 | `07-AI工具与技能/skills/` | **设计预期**，忽略 |
| 备份/归档 | `_archive/` `backup` | **设计预期**，忽略 |
| KC 知识卡片 | `KC-xxx` | **如果入链 < 3，考虑在相关工作流页面加链** |
| 导航/索引页 | `总览.md` `-总览.md` | **首页驾驶舱已挂载**，无需额外入链 |
| 报告/分析 | `方案.md` `分析.md` | **可能需补链**，在相关项目跟踪页加引用 |
