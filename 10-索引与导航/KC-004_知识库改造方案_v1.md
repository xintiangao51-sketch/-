## 二、我的修改建议

### 建议1：先修Claudian，再谈其他 ⭐关键依赖
Claudian当前故障。没有它，AI读写Obsidian的通道就断了。**建议作为第1优先级**。

### 建议2：00-99目录微调
```
🔴 02-业务中台 + 工程图纸 → 建议拆解：
   ├── 02-业务中台（招投标流程/资质/备案等流程性文档）
   └── 13-工程图纸（按项目名称子目录归档，配合Excalidraw标注）
   
🟡 09-项目资料 → 建议按年度分：
   ├── 09-项目资料/2026/
   ├── 09-项目资料/2025/
   └── ...（Dataview自动聚合，不依赖物理目录）
   
🔵 建议新增 00-工作台/Templates/ 独立目录
   （功能：存放Templater模板文件，和00-工作台日常笔记分离）
```

### 建议3：Module Obsidian（MOD）替代单纯PARA
你的00-99本质是PARA+自定义的混合体。引入**Module Obsidian (MOD)** 概念：
- 每个目录是一个"Module"（模块）
- 模块内统一包含3个文件：`INDEX.md`（导航）+ `README.md`（说明）+ `TEMPLATE.md`（模板）
- 模块间通过`KC-xxx <!-- placeholder -->`双向链接，MOC在10-索引与导航聚合

### 建议4：三阶段推进，不搞大爆炸改造
```
阶段1（基础设施）→ 阶段2（内容迁移）→ 阶段3（自动化）
避免一次性改造导致知识库中断使用
```

### 建议5：Kanban用Dataview替代（可选）
你已安装Kanban插件，但Dataview+Templater可实现动态看板：
```dataview
TABLE 项目名称, 截止日期, 状态
FROM "09-项目资料"
WHERE 状态 != "归档"
SORT 截止日期 ASC
```
可粘贴在看板笔记中，自动联动。

### 修复思路

2. **检查CLI路径** — cli-wrapper.cjs 和 codex.ps1 是否存在
3. **测试直连** — 跳过Claudian，直接用Claude Code CLI测试Obsidian库读写
