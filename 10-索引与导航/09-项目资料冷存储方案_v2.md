## 二、冷存储策略

### 2.1 分类规则

| 分类 | 文件类型 | 处理方式 |
|------|----------|----------|
| 🔵 冷数据 | PDF, DWG, JPG, PNG, TIF, ZIP, RAR | 已完结项目→移至冷存储 |
| 🟢 活跃数据 | XLSX, XLS, DOCX, DOC, MD, TXT, PY, CSV | 保留在知识库 |
| ⚠ 例外保留 | 造价成果PDF、合同扫描件PDF | 保留在知识库（工作产出） |

### 2.2 例外保留判定规则

不是所有PDF都该移走。以下PDF需保留在知识库：

| 保留类型 | 判定关键词 | 理由 |
|----------|-----------|------|
| 造价成果 | 预算、结算、清单、报价、招标控制价 | 工作产出，高频引用 |
| 合同文件 | 合同、协议、补充协议 | 法务文件，不可移走 |
| 签证变更 | 签证、变更、索赔 | 过程文件，可能复查 |
| 会议纪要 | 纪要、会议、备忘 | 文档类，体积小 |

**移走的PDF类型**：图纸PDF（无上述关键词的纯图纸扫描件）。

### 2.3 项目生命周期

```
项目阶段          存储位置                    文件保留
────────────────────────────────────────────────────────
在建项目          09-项目资料/                 全部保留
 ↓ 完工
已完结项目        09-项目资料/                 活跃文件 + 例外保留PDF
                  D:\冷存储\09-项目资料\        冷数据（图纸PDF/DWG/图片/ZIP）
 ↓ 3年后
长期归档          99-归档库/                   仅索引和成果文件
```

## 三、迁移脚本设计

### 3.1 脚本路径

```
D:\知识库\tools\cold_storage_migrate.py
```

### 3.2 核心逻辑

```python
"""
冷存储迁移脚本 v1
功能：
1. 扫描已完结项目的冷数据文件
2. 移动到 D:\冷存储\09-项目资料\{项目名}\{类型}\
3. 在原位置生成 _冷存储索引.md
4. 记录迁移日志

用法：
    python cold_storage_migrate.py --dry-run                    # 全部预览
    python cold_storage_migrate.py --dry-run --project "道班房"  # 单项目预览
    python cold_storage_migrate.py --project "道班房"            # 执行单项目
    python cold_storage_migrate.py --all                        # 执行全部已完结
    python cold_storage_migrate.py --rollback "道班房"           # 回滚单项目
"""

COLD_EXTENSIONS = {".pdf", ".dwg", ".jpg", ".jpeg", ".png", ".tif", ".tiff", ".zip", ".rar"}
EXCLUDE_KEYWORDS = ["预算", "结算", "清单", "报价", "招标控制价", "合同", "协议", 
                    "签证", "变更", "索赔", "纪要", "会议", "备忘"]
ACTIVE_PROJECTS = ["鼎梁筑", "鼎梁柱公司"]  # 进行中项目，不迁移

def is_cold_file(filepath):
    """判断文件是否为冷数据"""
    ext = filepath.suffix.lower()
    if ext not in COLD_EXTENSIONS:
        return False
    # PDF例外保留检查
    if ext == ".pdf":
        name = filepath.stem
        for kw in EXCLUDE_KEYWORDS:
            if kw in name:
                return False  # 例外保留
    return True

def migrate_project(project_dir, cold_storage_root, dry_run=True):
    """迁移单个项目"""
    # 1. 扫描冷数据文件
    cold_files = [f for f in project_dir.rglob("*") if f.is_file() and is_cold_file(f)]
    
    # 2. 按类型分类
    categorized = {"PDF": [], "DWG": [], "图片": [], "压缩包": []}
    for f in cold_files:
        ext = f.suffix.lower()
        if ext == ".pdf":
            categorized["PDF"].append(f)
        elif ext == ".dwg":
            categorized["DWG"].append(f)
        elif ext in (".jpg", ".jpeg", ".png", ".tif", ".tiff"):
            categorized["图片"].append(f)
        elif ext in (".zip", ".rar"):
            categorized["压缩包"].append(f)
    
    # 3. 生成索引
    index_content = generate_index_md(project_dir.name, categorized)
    
    if dry_run:
        # 只打印预览
        print(f"[DRY-RUN] {project_dir.name}: {len(cold_files)}个冷文件")
        return
    
    # 4. 创建冷存储目录
    cold_project_dir = cold_storage_root / project_dir.name
    for subdir in categorized:
        (cold_project_dir / subdir).mkdir(parents=True, exist_ok=True)
    
    # 5. 移动文件
    moved = []
    for subdir, files in categorized.items():
        for f in files:
            dest = cold_project_dir / subdir / f.name
            shutil.move(str(f), str(dest))
            moved.append({"src": str(f), "dest": str(dest), "size": f.stat().st_size})
    
    # 6. 写索引文件
    (project_dir / "_冷存储索引.md").write_text(index_content, encoding="utf-8")
    
    # 7. 记录日志
    log_entry = {
        "project": project_dir.name,
        "timestamp": datetime.now().isoformat(),
        "files_moved": len(moved),
        "total_size_mb": sum(f["size"] for f in moved) / 1024 / 1024,
        "files": moved,
    }
    append_log(cold_storage_root / "_索引" / "冷存储迁移日志.json", log_entry)
    
    return log_entry
```

### 3.3 命令行参数

| 参数 | 说明 |
|------|------|
| `--dry-run` | 预览模式，只打印不移动 |
| `--project "项目名"` | 指定单项目 |
| `--all` | 迁移所有已完结项目 |
| `--rollback "项目名"` | 回滚：从冷存储移回原位置 |
| `--kb-dir "路径"` | 知识库路径（默认D:\知识库） |
| `--cold-dir "路径"` | 冷存储路径（默认D:\冷存储） |

## 四、回滚方案

### 4.1 回滚触发条件

- 迁移后发现文件丢失
- Obsidian链接大面积断裂
- 找不到需要的文件

### 4.2 回滚流程

```
1. python cold_storage_migrate.py --rollback "道班房"
2. 脚本读取 _索引/冷存储迁移日志.json
3. 逐文件从 D:\冷存储\ 移回 D:\知识库\09-项目资料\道班房\
4. 删除 _冷存储索引.md
5. 验证文件数一致
6. 更新日志状态为 "rolled_back"
```

### 4.3 回滚安全保证

- 迁移日志记录每个文件的 src→dest 映射
- 回滚按日志逆序操作
- 回滚前验证冷存储中文件存在
- 回滚后验证文件数与迁移前一致
