## 二、冷存储策略

### 2.1 分类规则

| 分类 | 文件类型 | 处理方式 |
|------|----------|----------|
| 🔵 冷数据 | PDF, DWG, JPG, PNG, TIF, ZIP, RAR | 已完结项目移至冷存储 |
| 🟢 活跃数据 | XLSX, XLS, DOCX, DOC, MD, TXT, PY, CSV | 保留在知识库 |
| ⚠ 待定 | PDF(造价成果/合同文件) | 保留——这些是工作产出 |

### 2.2 项目生命周期

```
项目阶段          存储位置                文件保留
─────────────────────────────────────────────────────

                  D:\冷存储\09-项目资料\   冷数据移入
 ↓ 3年后
长期归档          99-归档库/              仅保留索引和成果文件
                  外部硬盘                 全部冷数据
```

## 三、迁移方案

### 3.1 迁移脚本设计

```python
# D:\知识库\tools\cold_storage_migrate.py
# 
# 功能：
# 1. 扫描已完结项目（无在建标记的项目）
# 2. 将冷数据文件移至 D:\冷存储\
# 3. 在原位置生成 _冷存储索引.md
# 4. 记录迁移日志
#
# 用法：
#   python cold_storage_migrate.py --dry-run    # 预览（不实际移动）
#   python cold_storage_migrate.py --project "道班房"  # 迁移指定项目
#   python cold_storage_migrate.py --all        # 迁移所有已完结项目
```

### 3.2 迁移优先级

| 优先级 | 项目 | 大小 | 理由 |
|:------:|------|------|------|
| P1 | 道班房 | 7.5GB | 最大项目，已完结 |
| P1 | 喀什体校 | 5.1GB | 已完结 |
| P2 | 青河G331 | 2.9GB | 已完结 |
| P2 | 新疆中科能源 | 1.9GB | 已完结 |
| P3 | 其他>500MB项目 | 各0.3-1.1GB | 按完结时间排序 |

### 3.3 预期效果

| 指标 | 迁移前 | 迁移后 | 变化 |
|------|--------|--------|------|
| 09-项目资料大小 | 32GB | ~7GB | -78% |
| 知识库总大小 | 63GB | ~38GB | -40% |
| Obsidian索引速度 | 慢 | 明显提升 | 文件数减半 |
| 冷存储大小 | 0 | ~25GB | 新增 |

## 四、风险与对策

| 风险 | 等级 | 对策 |
|------|:----:|------|
| 移动后找不到文件 | 🟡 | 每个项目留_冷存储索引.md，记录完整文件清单和路径 |
| Obsidian链接断裂 | 🔴 | 冷数据文件不参与Obsidian链接，活跃文件不动 |
| 误移活跃项目文件 | 🔴 | 迁移脚本先dry-run，人工确认后才执行 |
| 硬盘空间不足 | 🟡 | 冷存储可放外置硬盘，知识库留在D盘 |
| 嵌套目录混乱 | 🟡 | 迁移前先处理嵌套目录 |
