## 项目概述

搭建一个 Python 工程，实现：**图纸 → OCR识别 → 工程量提取 → 定额匹配 → 组价 → 报价表输出** 的全链路自动化。

## 详细规格

### Layer 1：图纸OCR识别与工程量提取

**输入**：PDF/图片格式的施工图纸
**输出**：`图纸工程量提取.xlsx`（部位、编号、尺寸、材质、数量）

**核心逻辑**：
1. PDF拆页 → 转图片（300DPI）
2. PaddleOCR 识别中文文字
3. 正则提取尺寸标注（如 `200×400`、`C30`、`Φ12@150`）
4. 按构件类型分类：梁/板/柱/墙/基础/钢筋
5. 自动计算工程量（长×宽×高 → 体积，面积等）
6. 输出结构化Excel

**关键代码要求**：
- 使用 `paddleocr` 库（中文识别精度高）
- 正则模式覆盖：截面尺寸、长度标注、材料标号、钢筋规格
- 支持批量处理整个文件夹
- 进度条显示

---

### Layer 2：清单自动组价

**输入**：工程量Excel（编码+名称+单位+数量+项目特征）
**输出**：`含单价报价表.xlsx`（GB50500-2013格式）

**核心逻辑**：
1. 读入工程量清单
2. 根据项目特征 → 匹配新疆2024定额子目
3. 套取定额人材机消耗量
4. 按费率计算：管理费28%、利润18%、规费22%、税金9%
5. 人工单价：130元/工日（伊犁）/ 147元/工日（喀什高海拔）
6. 输出综合单价 + 合价

**定额数据库** (`quota_db/xinjiang_2024.json`)：
需要预置常用的土建/安装定额子目，至少覆盖：
- 混凝土工程（现浇梁板柱墙）
- 钢筋工程（按规格）
- 砌体工程
- 模板工程
- 装饰装修

**关键代码要求**：
- 模糊匹配：项目特征关键词 → 定额子目
- 支持手动调整匹配结果
- 人材机分离显示
- 输出格式参照国标清单

---

### Layer 3：图纸清单核验

**输入**：图纸 PDF + 已有 Excel 工程量清单
**输出**：`差异核验报告.xlsx`（漏项/多算/偏差）

**核心逻辑**：
1. OCR 识别图纸工程量（同 Layer 1）
2. 读入已有清单
3. 逐项比对：编码相同 → 数量偏差；图纸有清单无 → 漏项；清单有图纸无 → 多算
4. 按偏差率着色（±5%以内绿色，±5-15%黄色，>±15%红色）
5. 输出差异汇总

---

### 主程序（main.py）

命令行菜单：
```
=== 鼎梁筑智能造价工作台 ===
1. 图纸OCR → 工程量提取
2. 工程量清单 → 组价报价
3. 图纸 vs 清单 → 核验差异
4. 一键全流程（图纸 → 报价）
0. 退出
```
