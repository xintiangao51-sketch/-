## 二、改进方向（按优先级）

### 🔥 P0：LLM集成（提炼增强）

**目标**：用LLM替代规则提取，提升知识卡片质量

#### 改进点1：auto_kc_generator.py LLM化

**当前问题**：
- 用关键词匹配分配领域，准确率低
- 摘要提取用简单正则，质量差
- 无法理解语义，漏掉重要内容

**改进方案**：
```python
# 伪代码
def extract_kc_with_llm(content, tags):
    prompt = f"""
    以下是知识库笔记内容，请提取结构化知识卡片：
    
    内容：{content[:1000]}
    已有标签：{tags}
    
    请输出：
    1. 领域分类（从定额/清单/工程量/组价/取费/人工/材料/机械/结算/变更/合同/施工/投标/案例中选）
    2. 一句话结论（≤30字）
    3. 关键数据（如有）
    4. 建议标签（3-5个）
    """
    return llm_call(prompt)
```

**实施步骤**：
1. 添加 `--llm` 参数
2. 批量处理时用LLM提炼
3. 保留原有规则作为fallback

**预期效果**：
- 知识卡片准确率从60%提升到90%+
- 自动生成高质量摘要
- 自动打标签

### 🔥 P1：语义检索（自动化检索）

**目标**：用向量检索替代关键词匹配

#### 改进点3：语义搜索引擎

**当前问题**：
- Obsidian搜索是关键词匹配，找不到语义相关的内容
- 断链检测无法推荐正确链接
- 孤岛笔记无法自动关联

1. **向量化全库**：用 bge-small-zh 生成Embedding
2. **语义搜索脚本**：`semantic_search.py`
3. **自动链接推荐**：在断链处推荐语义最相关的页面

**脚本设计**：
```python
# semantic_search.py
def semantic_search(query, top_k=5):
    """语义搜索引擎"""
    # 1. 加载向量索引（缓存）
    index = load_vector_index()
    
    # 2. 编码查询
    query_embedding = model.encode(query)
    
    # 3. 相似度检索
    results = index.search(query_embedding, top_k)
    
    # 4. 返回结果
    return results
```

**自动链接推荐**：
- 在 `fix_v15_real_broken_links.py` 中集成语义搜索
- 断链时自动推荐最相关的3个页面
- 生成修复建议报告

**预期效果**：
- 断链修复准确率提升
- 知识发现能力增强
- 孤岛笔记自动关联

## 五、风险与注意事项

1. **LLM成本**：
   - 本地模型（Qwen/DeepSeek）免费但质量略低
   - API调用（OpenAI/Claude）质量高但收费
   - 建议：先用本地模型，关键场景用API

2. **向量化性能**：
   - 1067篇笔记，首次向量化可能需要10-30分钟
   - 建议：增量更新，只处理新增/修改的笔记

3. **自动化边界**：
   - 自动化检测结果需要人工审核
   - 关键操作（删除/合并）需要确认
   - 保留undo机制（Git）
