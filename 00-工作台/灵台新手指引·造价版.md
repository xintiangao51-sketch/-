# 灵台新手指引·造价工程师版

> 5 分钟，把零散的造价资料（合同/签证/定额/价格）变成可检索、会生长的知识库。
> 基于灵台 LLM-Wiki 框架，造价场景本地化改造。

---

## 第 1 步：打开 D:\知识库

- Obsidian → 文件 → 打开 Vault → 选择 `D:\知识库`
- 不装 Obsidian 也能用，但装了图谱更直观

## 第 2 步：装 Python（如果没装过）

按 `Win+R`，输入 `cmd`，回车。在黑色窗口输入：

```
python --version
```

- 输出 `Python 3.x.x` → ✅ 跳过
- 提示"不是内部命令" → 装 [Python 3.10+](https://www.python.org/downloads/)（**勾选 Add to PATH**）

## 第 3 步：体检知识库

进入 D:\知识库 目录（或任何子目录），运行：

```
python D:\知识库\.scripts\lint_xinjiang.py
```

会输出一份体检报告：断链/死胡同/标签分布/品级分布/原料链路。
**目的**：先看清现状再动手。

## 第 4 步：扔第一份造价资料

打开 `D:\知识库\09-项目资料\` 找到对应项目目录，把造价资料拖进去：

- 一份已签约的施工合同 PDF（先转 Markdown）
- 一份签证单的扫描件（OCR 后转 Markdown）
- 一份结算审核意见 Excel
- 一段从《新疆造价信息》摘录的费率调整文字
- 一条突然闪过的造价灵感（新建 .md 写两行即可）

**格式**：Markdown 优先。Word/Excel/PDF 都能转，但 Markdown 检索最顺。

> 不会转？把 Word/PDF 内容复制粘贴，新建 .md 文件保存即可。

## 第 5 步：打开老高，说"提炼"

老高（Claude Code）会自动读取：

1. `D:\Claude-3p\CLAUDE.md`（身份配置）
2. `D:\知识库\10-索引与导航\AI-Agent五件套\`（六件套规则）
3. `D:\知识库\10-索引与导航\知识库总索引.md`（知识库导航）

然后对老高说：

> 提炼 [文件名或项目名]

老高会自己读规则、分类、归入对应知识域。

## 第 6 步：等着看

提炼完成后，去 `D:\知识库\01-造价核心\` 或对应专题目录看——老高把资料变成了知识页，自动加了标签和链接。

## 造价工程师专属建议

### 资料扔哪里？

| 资料类型 | 目标目录 | 标签建议 |
|---------|---------|---------|
| 施工合同 | `09-项目资料/[项目名]/合同/` | #合同 #法律 |
| 签证/变更 | `09-项目资料/[项目名]/签证/` | #签证 #变更 |
| 定额/取费 | `10-索引与导航/`（或 `01-造价核心/`）| #定额 #取费 |
| 材料价格 | `11-材料设备价格库/` | #材料 #价格 |
| 投标报价 | `05-投标报价/` | #投标 #报价 |
| 合同范本 | `14-标准模板/合同/` | #合同 #模板 |
| 法规/规范 | `08-参考规范/` | #法规 #规范 |

### 提炼的优先顺序

1. **高频引用资料**（标准合同、定额、费率表）→ 优先提炼
2. **新发布政策**（新建标〔2026〕xx 号、信息价）→ 24h 内提炼
3. **历史项目复盘**（已结项目的签证台账/结算单）→ 季度提炼
4. **碎片灵感**（费率/签证技巧）→ 随手提炼

### 不要做

- ❌ 一次性扔 50 份资料 → 提炼质量会塌
- ❌ 不写 frontmatter → 老高无法判断归属
- ❌ 把不同项目的资料混在一个文件 → 拆开
- ❌ 删除原始资料 → 知识库只追加 frontmatter，不改内容

## 常见问题

**Q: 老高提炼会改我的原始资料吗？**
A: 不会。`09-项目资料/` 是只读层，老高只在 01-07/14 等知识层写新页面。

**Q: 多久提炼一次？**
A: 每周 1-2 次，每次 5-10 份。连续扔 8 份以上触发 L2.5 批量摘要。

**Q: 提炼错了能回滚吗？**
A: 能。`D:\知识库` 是 Git 仓库（如果初始化了），`git revert` 一键回滚。

**Q: 不装 Obsidian 能看知识库吗？**
A: 能。VS Code + Markdown 插件，或直接用 Windows 资源管理器 + 文本编辑器。

**Q: 与 soul.md 是什么关系？**
A: soul.md 是老高的人格锚点（工程造价工程师身份），本文件是工作流指南，两者配合使用。
