# 故障恢复与成本控制规则

## 一、健康检查机制

| 检查项 | 触发条件 | 处理方式 |
|--------|---------|---------|
| Agent心跳 | 每任务结束 | 汇报progress + health状态 |
| 超时检测 | 单轮超过阈值无输出 | 标记⚠，连续3轮标记阻塞 |
| 质量检测 | 交付物评分<90 | 打回返工，列出扣分项 |

## 二、故障自动恢复

| 故障类型 | 检测方式 | 恢复策略 |
|---------|---------|---------|
| CC超时 | 无响应>阈值 | Hermes降级用轻量模型重试 |
| Codex执行失败 | 脚本报错 | 自动重试1次，失败转人工 |
| 任务失败3次 | 失败计数 | 标记blocked，人工介入 |
| 上下文溢出 | token接近上限 | 触发上下文压缩 |

## 三、模型分层策略

| 任务复杂度 | 推荐模型 | 说明 |
|-----------|---------|------|
| 深度判断（定额争议/合同条款） | CC (DeepSeek-v4-pro) | 需要推理链 |
| 格式整理（Excel/Word） | Codex | 速度优先 |
| 快速查询（信息价/定额编号） | 轻量模型 | 成本优先 |
| 批量处理（多文件扫描） | 子实例并行 | 并发3个 |

## 四、降级策略

```
正常 → 轻量降级 → 人工兜底
```

| 降级级别 | 触发条件 | 行为 |
|---------|---------|------|
| L0正常 | 无异常 | 标准流程 |
| L1轻量降级 | 主模型超时/报错 | 切换备用模型重试 |
| L2人工兜底 | 连续失败3次/红线操作 | 暂停任务，通知鼎梁筑 |
