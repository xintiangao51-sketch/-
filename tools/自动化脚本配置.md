# 自动化脚本配置

## lint_check_domain.py（适配18域）

### 12项检查
| # | 检查项 | 说明 |
|---|--------|------|
| 1 | 断裂链接 | 检查[[tools/自动化脚本工具箱]]指向是否存在 |
| 2 | 死胡同 | 无任何外链的页面 |
| 3 | 孤立文件 | 没有任何引用的文件 |
| 4 | 标签分布 | 标签使用统计 |
| 5 | 重复文件 | 同名文件 |
| 6 | 空目录 | 无md文件的目录 |
| 7 | frontmatter异常 | 缺少或格式错误 |
| 8 | 大文件 | >100KB的md文件 |
| 9 | 过期文件 | 30天未更新 |
| 10 | 命名规范 | 非法字符/空格 |
| 11 | 图片缺失 | 引用的图片不存在 |
| 12 | 引用统计 | 被引用次数排行 |

### 使用方法

`ash
# 全库检查
python tools/lint_check_domain.py

# 指定域检查
python tools/lint_check_domain.py 01-造价核心
python tools/lint_check_domain.py 09-项目资料
`

### 执行建议
- **日常检查**：Codex/定时任务
- **完整检查**：每周巡检

## check_log.py（提交前校验）

### 功能
- commit message vs 日志行数对比
- 确保变更与记录一致

### 使用方法
`ash
# 提交前自动运行
git add -A
python tools/check_log.py
git commit -m \"feat: xxx\"
`
